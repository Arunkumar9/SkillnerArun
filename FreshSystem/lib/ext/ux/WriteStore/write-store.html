Grids

There are several grids. Any cell in any grid can be double-clicked to edit it. The 'Save' button, if available, calls commitChanges() on the appropriate store. The 'Reject' button, if available, calls rejectChanges() on the appropriate store. Finally, the 'Undo' button, if available, rolls back the last individual change within this transaction. These buttons become available if there are any changes in the store that have not yet been committed or rejected.


These grids demonstrate success and failure of commitChanges(). All commits in this row pop up an Ext MessageBox to show the success or failure of the commit.

These grids demonstrate using one Store as the data source for another Store. The grid on the left is the list of persons. The one on the right is the addresses for the selected person. Select a person on the left to view their addresses on the right. Make a change to any address, then click 'Save' to update the store displayed on the left via a StoreProxy.
This grid demonstrates the effects of different write modes. The grid is not editable. When you click the "Execute" button below, changes will be made, and the information that would be transmitted to the server is shown in the appropriate box.
The changes are as follows, in order:

  3. Change the first record first name from "John" to "Jane"
